## 2024-05-20 - Cross-Site Scripting (XSS) via Attribute Breakout & javascript: URIs
**Vulnerability:** The custom `escapeHtml` function relied on assigning `textContent` to a DOM element and reading `innerHTML`. This only escapes `<`, `>`, and `&`, but leaves single and double quotes unescaped. As a result, an attacker could break out of HTML attributes (like `href` or `src`) and inject arbitrary events (like `onmouseover`). Additionally, the `href` attribute for `book.infoLink` was not validated to ensure it contained a safe protocol, allowing potential `javascript:` URI execution.
**Learning:** Browser DOM mechanisms like `textContent` -> `innerHTML` do not encode characters that are safe in text nodes but dangerous in attributes (quotes). Also, `escapeHtml` does not protect against malicious URL schemes in `href`.
**Prevention:** Use a robust `escapeHtml` implementation that explicitly replaces `&, <, >, ", '` with their HTML entities. For URL attributes, enforce an allowlist of safe schemes (like `http:` and `https:`).
